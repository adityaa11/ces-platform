## Workspace-cycle rejection

Temporary `cycle-a → cycle-b → cycle-a` workspace dependencies were introduced and checked with:

```bash
corepack pnpm install --lockfile-only
```

Observed result:

```text
ERR_PNPM_DISALLOW_WORKSPACE_CYCLES
There are cyclic workspace dependencies: packages/cycle-a, packages/cycle-b
Exit code: 1
```

The temporary packages were removed after the negative test.

## Phantom-dependency rejection

From `packages/requirement-schema`, an import of transitive package `picomatch` was attempted without declaring it:

```bash
node --input-type=module -e "import('picomatch')"
```

Observed result:

```text
Error [ERR_MODULE_NOT_FOUND]: Cannot find package 'picomatch'
Exit code: 1
```

This proves a workspace package cannot resolve that undeclared transitive dependency under the isolated linker.
